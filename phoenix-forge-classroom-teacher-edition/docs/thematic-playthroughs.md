# Thematic Playthroughs

Playthroughs group lessons into story-driven arcs. Teacher Edition uses the adult plan; Student Edition receives missions, badges, simple maps, and child-facing story prompts.

## Playthrough Design Rules
- Each playthrough should cover all seven top-tier categories.
- Each day should include one core lesson, one movement/body mission, one life-skill mission, one creative/maker mission, and one reflection.
- Story language should be exciting but not scary.
- The child should feel like a helper, builder, explorer, scientist, artist, caretaker, or community node.
- Teacher Edition stores metrics, supports, and observations.
- Student Edition shows maps, mission cards, check-ins, rewards, and short prompts.

## Playthrough 1: The Home Base Launch

Theme: Ezra prepares his home base for learning missions.

Core goals:
- Learn the daily map.
- Practice labels, counting, sorting, safety, body check-ins, and helper identity.
- Establish routines that the app can reuse every day.

Five-day arc:
- Day 1: Map The Base. Draw rooms, doors, learning zones, recovery hub, and material stations.
- Day 2: Label The Base. Label objects, tools, shelves, and containers.
- Day 3: Power The Base. Plan food, water, sleep, movement, and quiet time.
- Day 4: Repair The Base. Fix or organize one small thing.
- Day 5: Launch Review. Compare V1 and V1.1 of the learning space.

Student Edition badges:
- Map Maker.
- Label Decoder.
- Body Battery Engineer.
- Repair Helper.
- Home Base Captain.

- Maker pipeline.

Five-day arc:

- Day 2: Bridge Builder. Build a bridge for a toy.
- Day 3: Ramp Lab. Change ramp height and predict speed.
- Day 4: Tool Safety Trial. Practice ask/carry/use/return.
- Day 5: Show The Prototype. Present V1 and V2.

Student Edition badges:
- Prototype Builder.
- Test Pilot.
- Patch Master.
- Tool Safety Tech.
- Guild Builder.

## Playthrough Data Model

Each playthrough should store:
- Theme.
- Story premise.
- Age/stage.
- Target categories.
- Target subcategories.
- Daily itinerary blocks.
- Materials.
- Meals/snacks.
- Destinations.
- Student Edition badges.
- Teacher metrics.
- Supports likely needed.
- Evidence to collect.
- End-of-playthrough reflection.
