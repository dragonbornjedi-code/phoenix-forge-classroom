# Lesson And Activity Development Pipeline

Use this pipeline to build lessons for every subcategory in the Curriculum Taxonomy.

Default learner stage: five and a half. Lessons should feel like kindergarten missions: concrete, playful, movement-friendly, and short enough to succeed before fatigue or frustration takes over.

## Lesson Object

Required fields:
- Title.
- Top-tier category.
- Subcategory.
- Student-facing title.
- Age/stage.
- Duration.
- Energy level.
- Setting.
- Materials.
- Objective.
- Why it matters.
- Vocabulary.
- Setup.
- Steps.
- Questions to ask.
- Student Edition mission text.
- Evidence to collect.
- Progress metric.
- Independence scale target.
- Supports offered.
- Adaptations.
- Extension.
- Recovery option.
- Teacher reflection.

Optional fields:
- Book/media link.
- Destination.
- Food/snack tie-in.
- Outdoor tie-in.
- Maker/project tie-in.
- Student interest hook.
- Safety notes.
- Student Edition game link.

## Activity Variant Requirements

Every lesson should eventually have at least four variants:
- Visual: picture, card, map, diagram, schedule, matching.
- Physical: movement, build, sort, carry, trace, act out.
- Verbal/social: puppet, conversation, story, roleplay, explanation.
- Low-demand: quiet, short, sensory-friendly, no-fail version.

When useful, add:
- Outdoor version.
- Maker/tool version.
- Food/cooking version.
- Student Edition mini-game version.

## Lesson Quality Checklist

Before publishing to Teacher Edition:
- Objective is observable.
- Steps are short enough for a five-and-a-half-year-old.
- Materials are realistic.
- There is a built-in movement or sensory option.
- There is a clear success condition.
- There is a way to make it easier.
- There is a way to make it harder.
- There is a recovery plan if frustration rises.
- Adult notes stay out of Student Edition.

Before publishing to Student Edition:
- Mission title is playful.
- Instructions are short.
- Child sees only the next action.
- Reward/progress is simple.
- Help/break path exists.
- No adult analysis appears.

## Development Order

Phase 1: Taxonomy coverage.
- Create one starter lesson for every subcategory.
- Mark lessons as V0.
- Prefer household materials.

Phase 2: Daily itinerary integration.
- Attach starter lessons to Today blocks.
- Add meal, movement, rest, and transition supports.
- Track completion and friction.

Phase 3: Student Edition publishing.
- Convert strongest lessons to child-facing missions.
- Add icons, badges, and simple retry states.
- Import completion/check-in data back to Teacher Edition.

Phase 4: Evidence-based improvement.
- Review weekly data.
- Keep what works.
- Patch one variable at a time.
- Create V1, V1.1, V2 lesson versions.

## Batch Plan

For each top-tier category:
- Build 3 lessons per subcategory.
- One lesson should be movement-based.
- One lesson should be tabletop or maker-based.
- One lesson should be low-demand or recovery-friendly.

Minimum coverage target:
- Cognitive: 16 subcategories x 3 = 48 lessons.
- Emotional: 12 subcategories x 3 = 36 lessons.
- Practical: 14 subcategories x 3 = 42 lessons.
- Interpersonal: 12 subcategories x 3 = 36 lessons.
- Physical/Environmental: 12 subcategories x 3 = 36 lessons.
- Creative/Curiosity: 12 subcategories x 3 = 36 lessons.
- Values/Civic: 12 subcategories x 3 = 36 lessons.

Initial full library target: 270 lessons.
