# Memory Event Contract

To prevent architectural drift and inconsistent data entry, every feature in every Experience Shell must ingest data into the CMOS via a unified **MemoryEvent** pipeline.

## The MemoryEvent Structure
Every event generated by a shell (Student/Teacher) must adhere to this schema.

```json
{
  "eventId": "uuid-v4",
  "version": "1.0",
  "capturedAt": "timestamp-iso8601",
  "childAgeMonths": 65,
  "sourceShell": "STUDENT_APK_V1",
  "eventType": "DISCOVERY | BUILD | TRADITION | MILESTONE | REFLECTION | CONVERSATION",
  "narrative": {
    "title": "A short, descriptive title",
    "summary": "Brief summary of the occurrence",
    "childMood": "HAPPY | CURIOUS | TIRED | SAD"
  },
  "context": {
    "locationId": "landmark_backyard",
    "weather": "SUNNY",
    "involvedEntities": ["SPARK", "MOSS_WISP", "DAD"]
  },
  "assets": [
    {
      "id": "asset_id_01",
      "type": "PHOTO | AUDIO | PROJECT_3D",
      "localPath": "media/artifacts/2026/06/feather.jpg",
      "humanContributor": "DAD"
    }
  ],
  "impact": {
    "threadRef": "thread_naturalist",
    "wispAffinities": {"MOSS": 0.8, "VOLT": 0.2}
  }
}
```

## Ingestion Pipeline
1. **Emit:** The Shell (e.g., Student Edition) creates a `MemoryEvent` when an action occurs.
2. **Buffer:** The event is stored in a local "Drafts" table in CMOS.
3. **Stewardship:** The Teacher Edition displays the event for parent approval/annotation.
4. **Validation:** Once approved, the event is promoted to the **Living Chronicle** and **Artifact Registry**.
5. **PCAS Update:** The PCAS layer detects the new record and updates Spark's memory and the Museum layout.

## Rule of Ingestion
No Experience Shell may write directly to the `chronicle_episodes` or `legacy_artifacts` tables without passing through this event contract.
