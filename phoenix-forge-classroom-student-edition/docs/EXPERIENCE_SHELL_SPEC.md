# Student Edition: Adventure Shell Interface

The Student Edition is the primary **Experience Shell** for the child. It is a replaceable frontend that allows the child to interact with their world, participate in quests, and record their real-world discoveries.

## Core Mandates
1. **Experience First:** The child enters a world (**Hearthhome**), not a dashboard.
2. **Event Source:** The shell never writes to CMOS directly; it emits **MemoryEvents**.
3. **Identity Lenses:** The Wisps represent the child's own identity threads, reflecting growth visually.

## Primary Modules

### 1. Hearthhome Hub (The World as Menu)
- A 2D cozy landscape containing the **Museum (The House)**, the **Messenger Post**, the **Clock Tower**, and the **Garden**.
- Visuals reflect the time of day, season, and real-world weather.

### 2. Discovery Journal (The Capture Portal)
- The tool for recording reality.
- Prioritizes **Audio/Voice Capture** followed by Photography.
- Spark provides narration and encouragement during the capture loop.

### 3. The Quest Mirror (PCAS Interface)
- Displays "Missions" generated by the **PCAS Quest Engine**.
- Quests are narrative requests (e.g., "Find something blue" or "Build a stable bridge").
- Completion triggers a `MemoryEvent` with associated artifacts.

### 4. The Memory Shelf (Reflection)
- A visual display of the child's history.
- Tapping artifacts plays back the original human voices (Ezra, Mom, Dad).

### 5. Daily Rituals
- **Mood Monster:** Simple emotional check-in with Spark.
- **Victory Journal:** End-of-day voice-note reflection.

## Technical Execution
- Built as an Android APK (Kotlin/Compose).
- Consumes world state and quest definitions from the local **PCAS Interpreter**.
- Communicates with the Teacher Edition via the **Shared Sync Contract** to promote events from the buffer.
